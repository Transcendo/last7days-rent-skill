## 实现要求

- Pipeline：
  - profile -> query plan -> source candidates -> listing items -> privacy -> normalize -> risk -> dedupe -> scoring -> rerank -> render。
- Dedupe：
  - URL canonical。
  - 平台 ID。
  - 小区 + 户型 + 面积 + 价格。
  - 图片 hash。
  - 标题/正文相似度。
- Scoring：
  - hard filter。
  - commute/budget match。
  - trust score。
  - freshness score。
  - source preference。
  - risk penalty。
- Trust：
  - L0/L1/L2 自动产生。
  - L3 只由用户反馈或联系确认产生。
  - MVP 输入只接受 P0 来源；P1/P2/私域/WebSearch 候选必须被拒绝或标记为 out of scope。
- Report：
  - 聊天短名单 3-10 套。
  - Markdown 报告。
  - JSON evidence package。
  - 每套房必须有匹配理由、证据、不确定点、风险、下一步核验问题。
  - 输出 7 天行动计划。

## 不做的事

- 不调用地图实时路线 API。
- 不把热度作为主排序。
- 不把低风险写成已验真。
- 不输出联系方式明文。

## 验收标准

- fixtures 能离线跑完整 pipeline。
- 同源重复和多源重复都能聚合。
- P1/P2/私域/WebSearch 候选不会进入 MVP 排序和报告。
- 报告包含 7 天行动计划和隐私说明。
