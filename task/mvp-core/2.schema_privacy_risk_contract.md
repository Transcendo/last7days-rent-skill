## 实现要求

- `RentProfile` 必须围绕公司/办公点锚点建模，而不是只问城市。
- Profile 字段至少包括：
  - `profile_meta`
  - `privacy`
  - `user_goal`
  - `office_anchor`
  - `commute`
  - `housing_constraints`
  - `decision_preferences`
  - `scoring_weights`
  - `source_preferences`
  - `open_questions`
  - `provenance`
- `SourceCandidate` 用于 P0 source adapter 的中间候选，不接入 WebSearch、豆瓣、58/安居客 snippet、社媒链接或用户授权导入。
- `ListingItem` 用于结构化单源房源。
- `ListingCluster` 用于同一房源聚合。
- `VerificationEvidence` 用于官方核验入口、核验码、备案/平台链接。
- `FeedbackEvent` 用于结果反馈和 profile/source quality 调整。
- `privacy.py` 必须脱敏手机号、微信号、群名、真实姓名、头像/截图来源描述。
- `risk.py` 必须提供风险标签：
  - 低价异常。
  - 费用缺失。
  - 只给微信/电话。
  - 位置模糊。
  - 拒绝看房。
  - 催定金。
  - 私域或非 MVP 来源误入 pipeline。
  - P1/P2 来源不可进入 MVP listing pipeline。
- 缺失字段必须保持 unknown/None，不得补全。

## 不做的事

- 不实现 CLI。
- 不请求网络。
- 不保存真实用户隐私。
- 不输出正式报告。

## 验收标准

- schema 可被 Python 导入。
- 隐私测试证明手机号、微信号、群名、姓名不会进入公开输出。
- 风险测试覆盖 P1/P2 来源不可进入 MVP listing pipeline。
- 缺失价格、地址、押金、入住时间时不会被自动补全。
