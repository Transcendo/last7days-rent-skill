# 5. 最终集成与全链路校验

## Agent 角色

Final integration main agent。不要把本任务交给 subagent 完成；可以让 subagent 辅助排查，但最终判断必须由 main agent 汇总。

## 目标

整合所有并行 subagent 的结果，确认 `last7days-rent-skill` MVP 能安装、能创建本地 profile、能用 fixture/offline 或 P0 robot-friendly 公开源生成近 7 天候选报告，且所有 task plan 和隐私测试校验通过。

## 依赖

- 任务 1-4 全部完成。

## 必读上下文

- `README.md`
- `skills/last7days-rent/SKILL.md`
- `docs/product-contract.md`
- `docs/mvp-boundaries.md`
- `docs/architecture.md`
- `docs/dynamic-workflow-agents.md`
- `docs/goal-mode.md`
- `docs/integration-dry-run.md`
- `docs/release-checklist.md`
- 所有 `task/mvp-core/*.md`

## 实现要求

- 汇总每个 subagent 的 changed files、tests、warnings、risks。
- 检查写入范围是否被违反。
- 检查 multi-agent 是否被合理使用：如果某些任务被强行拆分反而增加冲突、重复实现或集成风险，需要在最终报告中指出并收敛为单 agent 串行修复。
- 检查 README、SKILL.md、CLI、schema、source、privacy、ranking、render、tests 是否一致。
- 检查实际目录结构符合 `docs/architecture.md`，且未创建非 MVP adapter。
- 检查本地 profile 默认写入 `~/.nesthub-rent/`，不是 repo。
- 检查报告默认写入 `~/.nesthub-rent/reports/`。
- 检查输出不含私人联系方式。
- 检查 slogan 和 7 天行动计划贯穿用户入口。
- 检查 MVP 只接入 P0 来源；P1/P2、私域、WebSearch、用户授权导入均未进入当前实现范围。

## 不做的事

- 不部署线上服务。
- 不发布包。
- 不自动提交 git。
- 不上传用户 profile 或报告。

## 集成检查

- Task plan 支持多 agent 并行执行。
- Task plan 同时支持“不适合 multi-agent 时单 agent 串行完成”的决策。
- Skill runtime 可以被 Agent 触发。
- CLI fixture 模式能生成报告。
- Profile 建档从公司/办公点开始。
- 信源只使用 P0 可信/robot-friendly 来源。
- P1/P2、私域、WebSearch、用户授权导入被明确排除在 MVP 外。
- 实际目录结构符合 `docs/architecture.md`。
- 给定 fixture profile 和 P0 fixture sources，能离线生成 Markdown 短名单和 JSON evidence package。
- L0-L3 语义正确。
- 隐私脱敏生效。
