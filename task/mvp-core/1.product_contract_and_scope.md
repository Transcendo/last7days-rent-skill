# 1. 产品契约、Slogan 与 MVP 边界

## Agent 角色

Main orchestrator agent。不要开启子 agent 执行本任务；本任务是所有并行任务的共享契约。

## 目标

建立 `last7days-rent-skill` 的 MVP 产品契约、repo 骨架和共享边界。所有后续 subagent 必须先读本任务产物再开始写代码。

核心 slogan 固定为：

```text
last7days = 帮助用户 7 天完成租房
```

MVP 定位固定为：

```text
近 7 天可溯源房源信号雷达：根据用户本地 profile，先从 P0 可信/robot-friendly 信源寻找候选房源，结构化、脱敏、去重、风险初筛、匹配排序，并输出 7 天租房行动计划和下一步核验问题。
```

## 实现要求

- 明确 MVP 能做什么：
  - 创建/读取本地 private profile。
  - 生成搜索计划。
  - 使用 P0 可信公开源。
  - 输出候选短名单、证据包、风险标签、下一步核验问题。
  - 输出 7 天租房行动计划。
- 明确 MVP 不承诺什么：
  - 不保证每套房仍在租。
  - 不保证发布者身份真实。
  - 不替代线下看房、付款、签约、合同审查。
  - 不自动抓私域。
  - 不绕验证码或登录墙。
  - 不接入 P1/P2 信源。
- 定义首批来源边界：
  - MVP 只做 P0：贝壳/链家/Ke 列表、Wellcee JSON-LD、房天下、官方核验入口。
  - P1/P2 统一靠边站，MVP 不实现：自如、我爱我家、58/安居客、小红书、微博、公众号、豆瓣、微信群/朋友圈/公司群/校友群、私域内容、WebSearch discovery、用户授权导入。
  - 后续阶段可以重新评估 P1/P2，但不得混入当前 MVP scope。
- 定义 L0-L3：
  - L0 原始线索。
  - L1 单源结构化。
  - L2 多源佐证。
  - L3 用户联系确认或明确反馈。
- 定义 repo-local 架构契约，写入 `docs/architecture.md`，后续所有 subagent 必须遵守。

- 所有文档必须强调本地 profile 和隐私脱敏。

## 不做的事

- 不实现业务代码。
- 不创建真实用户 profile。
- 不接入生产后端。
- 不抓取任何真实房源作为 repo fixture。
