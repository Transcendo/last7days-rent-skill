# 4. 集成 Dry Run 文档与执行记录

## Agent 角色

Integration-prep subagent。可与 `4.tests_fixtures_privacy.md` 并行执行。

## 目标

准备最终集成前的 dry run 文档和进度记录格式，让 main orchestrator 能汇总多个 subagent 的状态，并在 context 压缩或长流程中恢复工作。

## 依赖

- 任务 3 的 README/工作流文档至少初稿完成。

## 必读上下文

- `docs/dynamic-workflow-agents.md`
- `docs/goal-mode.md`
- `docs/subagent-handoffs.md`
- `docs/architecture.md`
- `README.md`

## 写入范围

只允许编辑：

- `docs/integration-dry-run.md`
- `docs/progress-ledger-template.md`
- `docs/release-checklist.md`

不要编辑测试和业务代码。

## 实现要求

- `integration-dry-run.md` 必须说明如何模拟一次完整用户流程：
  - 安装 skill。
  - 创建本地 profile。
  - fixture/offline 搜索。
  - 生成报告。
  - 给一个反馈。
  - 再次搜索或排序变化。
- `progress-ledger-template.md` 必须给出 subagent 状态记录格式：
  - task id。
  - owner agent。
  - write scope。
  - changed files。
  - tests run。
  - handoff notes。
  - warnings。
  - unresolved risks。
- `release-checklist.md` 必须包含：
  - README 检查。
  - Skill 安装检查。
  - CLI smoke。
  - pytest。
  - privacy scan。
  - task validator。
  - git status。

## 不做的事

- 不运行最终集成。
- 不修改业务代码。
- 不把文档写成泛泛流程，必须能被 final integration agent 直接执行。

## 验收标准

- 三个文档存在且步骤可执行。
- dry run 覆盖 profile -> search -> report -> feedback。
- progress ledger 能支持多 agent 长流程恢复。
