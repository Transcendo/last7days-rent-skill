## 写入范围

只允许编辑：

- `docs/troubleshooting.md`

不要编辑 SKILL.md、CLI、schema、source adapters、tests。

## 实现要求

- README 第一屏必须包含：
  - 产品名。
  - slogan：帮助用户 7 天完成租房。
  - 一句话定位。
  - 快速开始。
  - 非承诺边界。
- 安装命令必须包含：
  - 本地安装：`npx skills add /path/to/last7days-rent-skill -g`
  - 未来 GitHub 安装：`npx skills add <org>/last7days-rent-skill -g`
- README 必须包含至少 5 个可复制 prompts：
  - 自动接入/安装 prompt。
  - 首次创建 profile prompt。
  - 按 profile 搜近 7 天房源 prompt。
  - 对结果反馈并更新 profile prompt。
  - goal/multi-agent 执行 prompt。
  - P0-only MVP 执行 prompt。
  - goal/multi-agent prompt 必须说明：multi-agent 不是强制要求；如果任务耦合强或单 agent 更准确高效，应由 main agent 串行完成。
- README 必须列出：
  - 支持来源。
  - MVP 暂不支持的 P1/P2 来源。
  - 不支持/不绕过的来源。
  - 输出路径。
  - 隐私边界。
  - L0-L3 解释。
- `docs/prompts.md` 必须保存完整 prompt 模板。

## 不做的事

- 不夸大已完成能力。
- 不承诺 7 天内一定签约。
- 不把 CLI 写成主要用户入口。
- 不省略隐私和验证码边界。

## 验收标准

- README 首屏清楚说明“Agent Skill + 本地 private profile + 可信信源聚合”。
- 至少 5 个可复制 prompt 存在。
- README 中有 multi-agent goal mode prompt。
- README 的 multi-agent prompt 包含“不适合就不要强加”的判断准则。
- README 明确 MVP 只做 P0；P1/P2 先靠边站，不进入当前实现范围。
