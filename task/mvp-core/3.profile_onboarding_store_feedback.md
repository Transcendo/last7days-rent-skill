# 3. Profile 建档、决策题与反馈进化

## Agent 角色

Profile subagent。可与其他 `3.*` 任务并行执行；只处理 profile 写入范围。

## 目标

实现本地 private profile 的首次建档、公司/办公点锚点确认、MBTI 风格决策题、搜索计划编译和反馈进化。

- 任务 2 的 schema 和 runtime 完成。

## 实现要求

- `profile init` 第一问必须是公司/办公点/园区。
- 支持用户输入公司、办公点、地铁站、地址提示。
- 办公点解析不确定时必须让用户确认，写入 `open_questions`。
- 决策题必须是取舍题，不存“人格类型”标签，只存权重：
  - 通勤 vs 空间。
  - 可信来源 vs 低价线索。
  - 品牌公寓 vs 个人转租。
  - 速度 vs 尽调。
  - 安静安全 vs 生活便利。
- `profile-to-search-plan` 输出：
  - 城市/办公点/通勤圈。
  - 预算。
  - 租住方式。
  - source 优先级。
  - 风险过滤。
  - 需要追问的问题。
- `feedback` 支持：
  - 已租/失效。
  - 太远。
  - 太贵。
  - 疑似引流。
  - 不接受中介。
  - 来源不可信。
  - 真实可看。
  - 继续追踪。

## 不做的事

- 不上传 profile。
- 不查询用户公司内部系统。
- 不保存 cookie、私聊截图、他人联系方式。
- 不接入地图实时路线 API。

## 验收标准

- 本地 profile 可创建、读取、脱敏展示、删除/重置。
- 输入办公点不确定时生成确认问题。
- 决策题能更新 scoring weights。
- 反馈能写入 JSONL，并影响下一轮 mock ranking/source preference。
