# Product Contract

## Slogan

last7days = 帮助用户 7 天完成租房

## MVP 定位

近 7 天可溯源房源信号雷达：根据用户本地 profile，先从 P0 可信/robot-friendly 信源寻找候选房源，结构化、脱敏、去重、风险初筛、匹配排序，并输出 7 天租房行动计划和下一步核验问题。

## MVP 能做什么

- 创建、读取、脱敏展示本地 private profile。
- 首次建档从公司、办公点、园区开始，而不是从城市开始。
- 根据办公点锚点生成城市、通勤圈、预算、风险过滤和 source 优先级搜索计划。
- 使用 P0 公开可信来源：贝壳/链家/Ke 列表、Wellcee JSON-LD、房天下、官方核验入口。
- 结构化候选房源，保留字段 provenance，缺失字段保持 `unknown` 或 `None`。
- 脱敏手机号、微信号、私域群名、真实姓名、头像或截图来源身份线索。
- 去重聚合同一房源，生成 L0-L3 可信等级、风险标签和匹配理由。
- 输出聊天短名单、Markdown report 和 JSON evidence package。
- 输出下一步核验问题和 7 天租房行动计划。
- 记录用户反馈，用于下一轮 profile、source preference 和排序调整。

## MVP 不承诺什么

- 不保证每套房仍在租。
- 不保证发布者身份真实。
- 不替代线下看房、付款、签约和合同审查。
- 不自动抓私域内容。
- 不绕验证码、登录墙或反机器人机制。
- 不要求用户提供 cookie、token 或平台账号。
- 不接入 P1/P2 信源。
- 不用 LLM 补全未知价格、地址、押金、入住时间或联系方式。

## 可信等级

| 等级 | 含义 | MVP 处理 |
| --- | --- | --- |
| L0 | 原始线索，未结构化或来源不足 | 只进入候选池或被拒绝为 out of scope |
| L1 | 单源结构化，有原始 URL 或平台证据 | 可低置信展示，不能写成已验真 |
| L2 | 多源佐证或重复出现，关键字段基本一致 | 可进入短名单 |
| L3 | 用户联系确认或明确反馈仍在租、可看房 | 可重点推荐 |

L3 只能来自用户联系确认或明确反馈。官方核验入口只提供 `VerificationEvidence`，不能单独把列表房源提升为 L3。

## 硬性成功标准

给定一个 fixture profile 和 P0 fixture sources，离线生成一份 Markdown 短名单和 JSON evidence package。报告必须包含：

- profile 脱敏摘要。
- P0 source coverage。
- 候选房源短名单。
- L0-L3 可信等级。
- 匹配理由。
- 风险标签。
- 字段 provenance。
- 下一步核验问题。
- 7 天租房行动计划。

这条链路不依赖网络、不依赖真实用户数据、不接入 P1/P2。
